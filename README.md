# Corporate Sales Analytics and Tracking System

A dual-module Python implementation focused on rigorous logic profiling, localized data structure architecture, and an enterprise-grade CLI management system. This repository encompasses fundamental core algorithmic syntax alongside an optimized behavior replication matrix for transactional data auditing.

---

## Technical Architecture Overview

The codebase is logically decoupled into two standalone functional operational clusters to facilitate ease of modular unit testing and deployment separation.

### 1. Scripting Core Framework (`task1_exercises.py`)
This operational script addresses fundamental algorithmic constructs and computational metrics:
* **Variable Allocation & Casting Operations:** Strong-typed standard checking and formatted dynamic conversion routines.
* **Conditional Matrix Engineering:** Non-linear decision nesting loops designed to sort high-variance revenue metrics without relying on standard global paradigms.
* **Sequential Iteration Benchmarks:** FizzBuzz iteration parsing and standard linear-growth conditional evaluation algorithms.
* **Abstract Mathematical Routines:** Discrete localized functional structures for compound revenue estimations, weighted discount projections, and data array statistical balancing.

### 2. Transactional Console Engine (`sales_tracker.py`)
A production-ready CLI interaction paradigm optimized for dynamic localized storage configurations.
* **Persistence Mocking:** Implements cross-referenced composite mapping sequences (`dict` mapping to variable size continuous `list` objects).
* **Entity Deduplication:** Utilizes logical mathematical sets (`set`) to eliminate runtime taxonomy overlaps in category registries.
* **Fault-Tolerant Input Parsing:** Robust try-except structures wrapping raw console evaluation gates to intercept and isolate invalid data types before memory allocation.
